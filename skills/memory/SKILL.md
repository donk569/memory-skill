# Memory Skill — 跨对话记忆系统

记住对话内容，项目隔离。结束对话时保存结构化总结，新对话时手动加载恢复。

## 工作原理

### 对话中：标记重要内容

当用户表达"标记一下"、"记住这个"、"这个很重要"、"mark this"、"remember this"等类似意思时：

1. **立即在回复中确认**：简短告知用户"已标记"（避免打断对话流）
2. **内部记录标记内容**：把用户指向的讨论要点、决策、代码片段、文件路径记下来
3. **标记内容优先级最高**：保存记忆时，所有 ⭐标记内容必须出现在最终记忆文件中

### 对话结束时：保存记忆

当用户表达"这次对话结束了，帮我记下来"、"总结一下"、"save session"、"结束并保存"等类似意思时：

#### Step 1: 扫描对话，提取结构化信息

回顾整段对话，按以下字段提取：

| 字段 | 说明 |
|------|------|
| **对话摘要** | 1-2 句话概括本次对话做了什么 |
| **关键决策** | 做了什么决定，为什么。格式：`- 决策 → 原因` |
| **进行中任务** | 还没完成的事。格式：`- [ ] 任务描述` |
| **待解决问题** | 悬而未决的问题 |
| **下一步行动** | 下次应该做什么，清晰可执行 |
| **重要文件路径** | 涉及的关键文件。格式：`- \`path/to/file\` — 说明` |
| **⭐标记内容** | 用户主动标记的重要内容（必须包含） |
| **注意事项/坑点** | 踩过的坑、需要注意的陷阱 |
| **技术栈记录** | 相关框架、库、版本（如有） |

#### Step 2: 确定主题

- 优先问用户："这次对话用什么主题命名？（比如'修复登录模块bug'）"
- 如果用户说"随便"或未指定，根据对话内容自行推导一个简短主题

#### Step 3: 写入记忆文件

```markdown
# [对话主题]

**日期**: YYYY-MM-DD
**对话摘要**: 一两句话概括

## 关键决策
- 决策 → 原因

## 进行中任务
- [ ] 未完成任务

## 待解决问题
- 问题描述

## 下一步行动
- 下一步做什么

## 重要文件路径
- `path/to/file.ts` — 说明

## ⭐ 标记内容
- 用户强调的重要内容

## 注意事项/坑点
- 踩过的坑

## 技术栈记录
- 技术信息
```

保存路径：`.claude/memory/YYYY-MM-DD-主题.md`

#### Step 4: 更新索引文件

在 `.claude/memory/MEMORY_INDEX.md` 中添加一行链接：

```markdown
- [YYYY-MM-DD 主题](YYYY-MM-DD-主题.md) — 一句话摘要
```

如果 `MEMORY_INDEX.md` 不存在，创建它，开头写：

```markdown
# 对话记忆索引

按时间倒序排列，点击加载对应记忆。

---
```

新记忆添加到列表最前面。

### 新对话开始时：加载记忆

当用户表达"加载记忆"、"继续上次的"、"load memory"、"恢复上下文"、"上次做到哪了"等类似意思时：

#### Step 1: 检查是否存在记忆

读取 `.claude/memory/MEMORY_INDEX.md`：
- 如果不存在或为空：告知用户"当前项目没有保存的记忆"
- 如果存在：列出所有记忆文件

#### Step 2: 展示记忆列表

```
📂 当前项目记忆：

1. 2026-06-21 实现跨项目记忆技能 — 设计了memory skill的需求和架构
2. 2026-06-20 修复登录模块过期token问题 — 修复了auth中间件的bug
3. 2026-06-18 重构用户模块 — 拆分UserService为多个子模块

输入编号（1-3）加载对应记忆，或输入"最近"加载最新。
```

#### Step 3: 加载选中的记忆

读取对应文件，将内容注入当前对话上下文。然后告知用户：

```
✅ 已加载记忆：2026-06-21 实现跨项目记忆技能

关键摘要：设计了memory skill的需求和架构
进行中任务：实现SKILL.md、创建GitHub仓库
下一步：完成skill编写并发布到GitHub
```

此后用户可以直接继续工作，无需重复解释上下文。

## 存储规范

```
<项目根目录>/.claude/memory/
├── MEMORY_INDEX.md                 # 索引文件，列出所有记忆
├── 2026-06-21-实现跨项目记忆技能.md
├── 2026-06-20-修复登录模块bug.md
└── ...
```

- 每个项目独立，互不影响
- 文件名格式：`YYYY-MM-DD-主题.md`
- 索引文件始终在第一位（按字母序）

## 重要原则

1. **标记内容绝对不丢**：用户标记的 ⭐内容必须在最终记忆中出现
2. **简洁但不遗漏**：每个字段的信息精简，但关键信息不能省
3. **可执行**：下一步行动要具体到"打开哪个文件、写什么代码"，不是"继续开发"
4. **不自己触发保存**：只在用户明确表达结束意图时才保存，不要主动替用户决定
5. **项目隔离**：只操作当前项目的 `.claude/memory/`，不要跨项目读写
